OOP(Object oriented programming)

It is a programming paradigm that is based on the concept of "objects" which can contains data and code that manipulates the data.

Key concept of OOPs:-
1.Class:- A class is a blueprint for creating object that encapsulate data and methods.
-Class can be created by "class" keyword
-A class Can have constructor (__int__) and define their own behavior via methods.
Syntax: class className:
  constructor(self, parameters):
  self.variable = parameter
  def methodName(self):
  instructions

2. Object: An object is an instance of a class.
-Objects have attributes/properties and method.
-We can create multiple objects from the same class, each with their own attribute values.
Syntax:-
  object1 = className(arguments)

3.Methods :- Methods are functions defined inside a class that describes the behaviors of an object.
-All methods have "self" keyword as their first parameter, which allows them to access and modify the object's attribute.
Syntax:- Class className (self):
  iinstructions

4. Self keyword :-Self refers to the current object or current instance of the class.
-It enables each object to maintain it's own attribute values.
-Without self keyword, python would not know which object's data to use in methods or constructor.

5. Constructor:-A constructor is a special method which is called automatically when an object is created from a class.
-It is used to initialize the object's attribute or set initial state.
__init__ is the constructor in python, which is defined inside the class.
Syntax:-
  class className:
  constructor(self, parameters):
  self.variable = parameter

Types of Constructor:-
a> Default Constructor:- It takes no arguments except ‘self’ keyword.
b> Parameterized Constructor:- It takes one or more parameters including self.
-It initializes objects attribute with user value.

Polymorphism:
-It means "many forms". It allows a function or method with the same name to perform differently depending on the type of object.
e.g: class Dog:
  def speak(self):
  print("Woof")
class Cat:
  def speak(self):
  print("Meow")
dog1 = Dog()
dog1.speak()
cat1 = Cat()
cat1.speak()

Encapsulation:- Encapsulation is an OOP principle that involves binding data and methods within a class and restricting direct access to some of the class's components.
-It improves security, and maintainability of code by controlling how data is accessed and updated.
-We can achieve encapsulation by converting public variables into private variable by using double underscore(__) before the variable name.
-It Keeps our code organized and secure by ensuring that objects protect their own data and expression only what is necessary.
eg: class Bank:
  def __init__(self, account , balance):
  self.__balance = balance
  self.account = account
  def desposit(self,amount):
  if amount > 0:
  self.__balance += amount
  else:
  print("Invalid amount")
  def withdraw(self,amount):
  if amount > 0 and amount <= self.__balance:
  self.__balance -= amount
  else:
  print("Invalid amount")
  def get_balance(self):
  return self.__balance
account1 = Bank("123456", 1000)
print(account1.get_balance())
account1.desposit(100)

account1.withdraw(500)

Error:-
Errors in programming referred to bugs or problems within a program that prevents it from functioning.
-There are 4 types of errors in Python:-
 	1)Syntax Error :- Mistakes in code structure like missing semicolon, spelling mistake for keywords, which detected before execution.
 	2)Runtime Error :- Occurs while code runs, such as ZeroDivisionError.
 	3)Logical Error :- Code runs but produces unexpected results due to wrong logic.
 	4)Indentation Error :- Occurs due to incorrectly indented code blocks.
-Errors in software or program is called bugs, and fixing errors or bugs is called debugging.
-Some common errors are:- TypeError, ZeroDivisionError, ImportError, etc.

Exception: An exception is a special type error that interrupts normal flow of program.
-When an error occurs, Python creates an exception object that contains details about the error or problem, and then throws it to stop the execution.
-Python includes many built-in exception classes, all are derived from Exception.

Exception Handling: Exception handling separates error detection from business logic using try-except-finally blocks.
 	try block:- We write codes in try block which may occurs exception or error.
 	except block:- Handles exceptions, also we can specify specific type for response.
 	finally block:- It runs always, whether there is exception or not.
Syntax:-
try:
 	//instruction
except:
 	//instruction
finally:
 	//instruction
-We can specify specific and multiple exception block within a program, which makes programs safer and easier to debug, and allows custom responses to differ error types.

##NumPy: NumPy is a powerful python library for numerical computing which is known for its fast, versatile and advanced mathematical capabilities
-NumPy is an open-source library developed to perform numerical operations on large, multidimensional array and matrices
-NumPy stand for numerical Python
-It supports various mathematical algebraic and statistical functions making it easy for data science and machine learning
-NumPy arrays are known as nD arrays representing homogenous collection of data delivering up to 50 times faster than python lists.

# Difference between array and python list
NumPy arrays called homogeneous type of data while list calls heterogeneous type of data.
NumPy arrays is much faster for large data-set due to its memory layout but lists are slower because of their dynamic structure.
NumPy arrays supports advanced mathematical which is compare with list.
NumPy arrays consume less memory as they are stored in contiguous memory blocks.
lists are mutable but NumPy arrays are immutable.
NumPy arrays are used for numeric computation while python lists are used for general purpose only.

-itemSize Operator:  It gives the size in bytes of each element in a NumPy array determined by its datatype.
-dtype Operator: It returns the datatype of a NumPy array like int32 float64 etc.
-size Operator: It returns the total numbers of elements in the array
-shape Operator: It returns a tuple indicating the size of the array along each dimension. ex: (2,3)

##Array Creation and basic manipulation Methods
